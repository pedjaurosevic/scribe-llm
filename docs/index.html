Three guarantees

1The tool call cannot break

On a llama.cpp server, Scribe builds a grammar from your tool schemas and constrains the model with it, so a malformed tool call is grammatically impossible — the right tool name, the required arguments, valid JSON. If a model still fumbles, the call is re-asked under the grammar.

How: GBNF (llama.cpp's grammar-constrained decoding — a proven technique) auto-generated from the tool schemas and wired as a tool-call integrity layer. Guarantees the call's form, not the model's judgment.

2Cite or refuse

Grounded Q&A maps every claim to a numbered source [n], tags [CONTRADICTION] when sources disagree, and refuses to answer outside the sources instead of hallucinating.

How: hybrid retrieval (SQLite FTS5 + vectors, fused with Reciprocal Rank Fusion) feeds numbered sources into a grounding prompt.

3Grounding is measured, not claimed

scribe bench reports a deterministic Source-Presence Index (SPI) over a checksum-locked held-out suite. On Gemma 4 12B it scores SPI 1.00 — and you can re-run it on your own model.

How: SPI scores citation coverage on answerable tasks and correct refusal on impossible ones — no LLM judge, fully reproducible.

Honest scope: GBNF and constrained decoding are well-known (llama.cpp added grammars in 2023; the same idea ships as "structured outputs" elsewhere). Scribe's contribution is the integration — using it as an automatic tool-call safety net for small local models — not the technique itself. Guarantees 1–3 hold on llama.cpp; other backends degrade gracefully to a best-effort text parser.

What it does

Universal LLM adapter

Point it at any OpenAI-compatible endpoint — local llama.cpp, Ollama, LM Studio, or remote. scribe discover finds them.

Safe code mode

/code with a destructive-command gate, a Python AST gate, a bubblewrap sandbox, and git checkpoint / rollback.

Cross-session memory

SME (Semantic Memory Engine) recalls your last session, and a WorldModel persona keeps the agent's identity stable across runs.

Hybrid RAG over your docs

FTS5 + local embeddings (multilingual-e5) fused with RRF — exact identifiers and paraphrase both retrieve.

Observability

Deterministic ORORO session traces and a machine-readable scribe status --json contract.

Rich TUI & web UI

A terminal interface (Rich + Textual) and a streaming FastAPI web chat with login.

Project vaults

scribe init gives any directory its own isolated RAG / SME stores.

Blind compare

scribe compare A/B tests two models on one prompt without telling you which is which — pick a local model honestly.

Modular skills

Deep-research, writer, and wiki-memory skills — drop in more as SKILL.md modules.

Quick start

# 1. Clone & install
git clone https://github.com/pedjaurosevic/scribe-ai.git
cd scribe-ai
./scripts/install.sh        # installs the package + config + ~/scribe-workspace

# 2. Start your llama-server (or use any OpenAI-compatible endpoint)
./scripts/start-server.sh

# 3. Find your model server, then chat
scribe discover             # scan local ports (+ --tailscale)
scribe chat                 # … or: scribe web  → http://localhost:8765

# 4. See the grounding number for yourself
scribe bench                # judge-scored fitness + deterministic SPI

Configure the endpoint in ~/.config/scribe/config.toml or via SCRIBE_BASE_URL / SCRIBE_MODEL env vars.
